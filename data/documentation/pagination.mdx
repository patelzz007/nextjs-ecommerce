# Pagination

All list endpoints support cursor-based pagination to efficiently navigate through large result sets.

## Using Pagination

Use the `limit` and `cursor` parameters:

\`\`\`bash
curl "https://api.yourcompany.com/v1/products?limit=20" \
  -H "Authorization: Bearer YOUR_API_KEY"
\`\`\`

\`\`\`javascript
const response = await fetch(
  'https://api.yourcompany.com/v1/products?limit=20',
  {
    headers: {
      'Authorization': `Bearer ${YOUR_API_KEY}`
    }
  }
)

const data = await response.json()
console.log(data.next_cursor)
\`\`\`

\`\`\`python
import requests

params = {'limit': 20}
response = requests.get(
  'https://api.yourcompany.com/v1/products',
  headers={'Authorization': f'Bearer {YOUR_API_KEY}'},
  params=params
)

data = response.json()
next_cursor = data.get('next_cursor')
\`\`\`

## Pagination Response

\`\`\`json
{
  "data": [...],
  "has_more": true,
  "next_cursor": "eyJpZCI6MTIzfQ=="
}
\`\`\`

## Fetching Next Page

Use the `next_cursor` value to fetch the next page:

\`\`\`bash
curl "https://api.yourcompany.com/v1/products?limit=20&cursor=eyJpZCI6MTIzfQ==" \
  -H "Authorization: Bearer YOUR_API_KEY"
\`\`\`

## Pagination Limits

- Default limit: 10 items
- Maximum limit: 100 items
- Cursors expire after 24 hours
